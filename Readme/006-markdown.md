# Todo List
## 專案介紹
透過開發 Todo List 練習使用 JavaScript 建立互動式網頁功能， 並從使用者角度思考待辦事項管理流程，驗證功能是否符合預期需求， 同時培養問題分析、功能驗收與資料同步的實作能力
---
## 功能
- 新增待辦事項
- 刪除待辦事項
- 標記完成
---
## 開發過程

1. 動態新增元素無法觸發事件
### 問題

新增待辦事項後，刪除按鈕無法正常運作。

### 原因

一開始直接對按鈕綁定 click 事件，但這些事件只會綁定在頁面載入時已存在的元素上。後續透過 JavaScript 動態新增的按鈕不會自動繼承事件監聽。

### 解決方式

改用 Event Delegation（事件委派），將事件監聽綁定在待辦事項列表父層，再透過 event.target 判斷實際被點擊的元素，使新增的項目也能正常觸發事件。

2. 功能判斷衝突
### 問題

新增完成按鈕後，點擊完成按鈕時也會誤觸刪除功能。

### 原因

一開始透過元素標籤名稱（button）判斷點擊目標，但完成按鈕與刪除按鈕皆為 button 元素，因此無法正確區分功能。

### 解決方式

改用 class 名稱判斷不同按鈕的用途，分別處理完成與刪除功能，避免事件判斷衝突。

3. 輸入驗證與使用者體驗優化
### 問題

使用者只能透過滑鼠點擊新增按鈕建立待辦事項，且輸入空白內容時仍可建立任務。

### 原因

功能初期僅監聽 click 事件，且未驗證輸入內容是否有效。

### 解決方式

將新增邏輯獨立封裝成函式，同時監聽 click 與 Enter 鍵事件，提高操作便利性。另外利用 trim() 去除前後空白，若結果為空字串則停止新增，避免建立無效資料。

4. 資料持久化與同步
### 問題

重新整理頁面後，待辦事項資料消失，且初期只能保留部分資料。

### 原因

資料僅存在 JavaScript 執行期間的記憶體中，重新整理後即被清除。另外 localStorage 只能儲存字串，無法直接保存陣列或物件資料。

### 解決方式

使用 localStorage 保存待辦事項資料，儲存時透過 JSON.stringify() 將資料轉換為字串，讀取時使用 JSON.parse() 還原資料。同時在新增、刪除與完成狀態切換後同步更新 localStorage，確保畫面與資料保持一致。
