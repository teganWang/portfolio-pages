Todo List

透過開發 Todo List 練習使用 JavaScript 建立互動式網頁功能， 並從使用者角度思考待辦事項管理流程，驗證功能是否符合預期需求， 同時培養問題分析、功能驗收與資料同步的實作能力。
功能特色:
．待辦事項新增與刪除
．待辦事項完成狀態切換
．資料與畫面同步更新
．防止新增空白代辦事項
技術實作:
．使用 JavaScript 操作 DOM 元素
．使用事件監聽處理使用者互動
．使用 Event Delegation 管理動態產生的按鈕
．練習資料狀態與畫面同步邏輯
．將功能拆分為獨立函式提升可維護性
產品與使用者體驗思考:
．規劃待辦事項新增、完成與刪除流程
．設計輸入驗證機制避免建立無效資料
．驗證功能是否符合預期操作需求
測試與驗證:
．點擊新增按鈕可正確新增待辦事項
．按下Enterr鍵可快速新增待辦事項
．點擊刪除按鈕可正確刪除待辦事項
．輸入框空白時不建立待辦事項
．點擊完成按鈕可正確接換狀態並更新樣式
．確認資料與畫面顯示的內容保持一致
接下來預計：
．Local Storage 資料持久化
．代辦事項篩選功能
．Dark Mode 深色模式

購物車數量加減元件

模擬電商網站中的商品數量調整功能，練習使用 JavaScript 實作使用者互動與資料同步邏輯。
功能特色:
．商品數量增加與減少
．即時更新畫面顯示結果
．防止商品數量低於 0
．支援長按連續加減操作（開發中）
技術實作:
．使用 JavaScript 操作 DOM 元素
．使用事件監聽處理使用者互動
．練習資料狀態與畫面同步更新
．將功能拆分為獨立函式提升可維護性
產品與使用者體驗思考:
．考量使用者快速調整商品數量的需求
．設計數量下限機制避免產生無效資料
．驗證不同操作情境下功能是否正常運作
測試與驗證:
．點擊增加按鈕可正確增加數量
．點擊減少按鈕可正確減少數量
．數量為 0 時不可繼續減少
